# Experiment Canvas

> **Note**: This template is now fully integrated into SKILL.md (v2.0+). This file is kept as a standalone reference.

Designed to test the riskiest assumption of a new concept with minimal resources.

| Field | Description |
|---|---|
| **Concept Name** | [Name] |
| **Core Hypothesis** | [What must be true for this concept to work?] |
| **Riskiest Assumption** | [The one most likely to kill the idea] |
| **Experiment Description** | [What will you do?] |
| **Minimum Success Criteria** | [What numbers would prove it viable?] |
| **Required Resources** | [Money, time, materials, people] |
| **Measurable Metrics** | [What you'll track] |
| **Stop / Pivot / Continue Rule** | [Decision thresholds] |
| **Estimated Time to Result** | [Duration] |

## Example
| Field | Value |
|---|---|
| Concept Name | Battery swapping stations for electric scooters |
| Core Hypothesis | Riders will accept swapping instead of charging if it takes under 2 minutes |
| Riskiest Assumption | People won't want to "rent" a battery they don't own |
| Experiment | Set up a free swap station at one high-traffic corner, no commitment |
| Minimum Success Criteria | 20% of passing scooters stop to swap in week 1 |
| Required Resources | 10 batteries, 1 swap machine mockup, 1 attendant |
| Metrics | Swap count, repeat users, NPS survey |
| Decision Rule | < 5% adoption → kill; 5-20% → iterate; > 20% → invest |
| Time to Result | 2 weeks |
